Settings menu

The settings menu is a separate floating window so it can be opened independently from the editor. This allows for people to fix settings which crash the editor on startup.

Startup arguments

There are a couple of startup arguments, if non are given it will default to 'type settings'.

  • --type=<settings, project_settings>: This decides which settings menu is displayed;
  • --project_path=<path to a project file>: Needed when type is project_settings;

Layout builder

The way the layout builder work is by using the settings menu config or the correct type (settings/project_settings). We loop through each section and each section gets a section label, followed by all settings labels and their nodes to change their values, and closed of with a separator.

Adding a setting

Inside of the editor you can add a setting in the settings manager, for custom modules there will be a separate method. We need a setter, a getter and a meta getter. Naming scheme for these are 'set_*setting_name*', 'get_*setting_name*', and 'get_*setting_name*_meta'.

Localization needs to be taken into account and setting entries should also be added to the translations with following keys: 'SETTING_*setting name all caps*' and 'TOOLTIP_SETTING_*setting name all caps*'.

Meta

The meta data for each setting should contain a type, which can be 'text', 'float', 'int', 'bool', ... all lower case. Except for type, you also need a 'default' entry for the default value of the setting.

For int and float you'll need a 'step', 'min_value', and 'max_value' as well. If you want to create an OptionButton you'll need to have a 'options' entry which is a dictionary with it's structure like this: {key, option_name}.
